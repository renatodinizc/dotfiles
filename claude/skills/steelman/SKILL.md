# Steelman Protocol

Build the strongest possible case for a position. The output must pass the **Ideological Turing Test**: a genuine believer of that position would read it and nod along, not say "that's a strawman."

## Process

### 1. Identify the Position
State the position being steelmanned clearly. If it was being dismissed, note what the dismissal argument was.

### 2. Find the Core Logic
Answer: **Why would a smart, well-informed person hold this position?**
- What evidence supports it?
- What values or priorities make it rational?
- What does this position get right that the opposing view misses?

### 3. Build the Case (3-5 key arguments)
For each argument:
- Lead with the strongest version, not the weakest
- Use concrete evidence, numbers, and examples
- Show the internal logic, not just external claims
- Address the most obvious objection preemptively

### 4. The Compression Hierarchy
Present the steelman in this order:
1. **The crux**: The single point that matters most (1-2 sentences)
2. **The concrete case**: Numbers, scenarios, named risks (1 paragraph)
3. **Why a smart person believes this**: The internal logic and values (1 paragraph)

### 5. Honest Assessment
After presenting the steelman, add a brief note:
- What the steelman gets right that is genuinely worth considering
- What the strongest remaining objection is even after the steelman
- Whether the steelman changes the overall assessment

## When NOT to Steelman
- Empirically settled questions (evolution, climate science, vaccine efficacy)
- Rationalizations for clearly self-destructive behavior
- Positions the user has already heard, considered, and addressed
